# Quickstart: Quran Reading Modes

**Branch**: `008-quran-reading-modes` | **Date**: 2026-03-13

## Prerequisites

- Flutter 3.x with Dart 3.10.4+
- Existing Mesk Islamic App codebase on `008-quran-reading-modes` branch
- Android/iOS device or emulator for testing

## Key Files to Understand First

### Data Layer (read these first)
1. `lib/features/quran/data/services/quran_database_service.dart` — SQLite setup, table creation, seeding
2. `lib/features/quran/data/datasources/quran_db_datasource.dart` — All verse/juz queries
3. `lib/core/services/storage_service.dart` — SharedPreferences wrapper with existing Quran keys

### Presentation Layer
4. `lib/features/quran/presentation/screens/quran_reader_screen.dart` — Current PageView-based Mushaf reader (becomes Full Quran mode)
5. `lib/features/quran/presentation/widgets/mushaf_page_view.dart` — Per-page verse rendering
6. `lib/features/quran/presentation/widgets/verse_widget.dart` — Individual verse display with actions
7. `lib/features/quran/presentation/providers/quran_provider.dart` — Reader state + data providers

### Theme & Design System
8. `lib/core/constants/app_colors.dart` — Color palette (primaryAccent #7C3AED)
9. `lib/core/constants/app_text_styles.dart` — Typography (Amiri for Arabic, Nunito/DM Sans for English)
10. `lib/core/widgets/clay_card.dart` — Primary card component

## Architecture Pattern

```
Feature follows clean architecture:

data/
  datasources/    → Direct data access (SQLite, JSON, API)
  models/         → Data transfer objects
  repositories/   → Repository implementations
  services/       → Database initialization

domain/
  entities/       → Business objects
  repositories/   → Abstract contracts
  usecases/       → Business logic (optional)

presentation/
  providers/      → Riverpod state management
  screens/        → Full-screen widgets
  widgets/        → Reusable UI components
```

## Critical Context

1. **Only 12 sample verses exist** in `quran_verses.json` — full 6,236 needed before any meaningful testing
2. **Page persistence is broken** — `StorageService.setLastReadPage()` exists but `QuranReaderNotifier` never calls it
3. **Tafsir is stubbed** — `TafsirBottomSheet` shows placeholder text only
4. **No translation data** — neither in DB schema nor assets
5. **readingMode key exists** — `StorageService.readingMode` defaults to `'mushafPage'`, already in `SettingsState`
